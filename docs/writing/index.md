# Writing

This section is meant to replace the half-finished Blogspot workflow with something easier to maintain.

Instead of drafting posts in a separate editor and forgetting where the final version lives, each article here can be a markdown file in the same repository as the site. That means:

- version history is automatic,
- publishing is just a git push,
- and project pages can link directly to deeper notes without duplicating content.

## Migration Plan

- Keep the old Blogspot posts as archive links.
- Publish new and revised long-form posts here first.
- Cross-link each new article from LinkedIn, YouTube descriptions, or GitHub repos.
- Use Blogspot only as a legacy archive if you still want it online.
