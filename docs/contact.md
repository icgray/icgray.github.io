## Recommended Publishing Workflow

1. Keep the website in its own GitHub repository.
2. Write project pages and articles as markdown in `docs/`.
3. Link each project page to the matching GitHub repo, video demo, and any paper or slides.
4. Let GitHub Actions rebuild and publish the site on every push.

This gives you one stable URL that can point outward to GitHub, LinkedIn, Blogspot, and YouTube while still letting GitHub be the source of truth for updates.
